# Native shell clients

В этой папке вынесены отдельные нативные клиенты, чтобы не смешивать их с основным Next.js фронтендом.

## Структура

- `electron/` — Electron-клиент для Windows
- `tauri/` — Tauri-клиент для Windows и Android

## Текущая архитектура

Оба клиента работают как нативные оболочки над опубликованной удалённой веб-версией приложения.

Это выбранный безопасный способ переноса без ломки существующего Next.js приложения:

1. основной веб-фронтенд продолжает жить отдельно;
2. Electron/Tauri открывают удалённый URL приложения в нативном окне;
3. опубликованный веб-фронтенд уже подключается к удалённому FastAPI-бекенду.

## Что нужно указать

В обоих клиентах настройте `REMOTE_WEB_URL`.

Пример:

- `https://vokrug-sveta.shar-os.ru`

Если позже потребуется полноценный нативный UI без webview-оболочки, следующим шагом можно будет вынести общий API-слой и постепенно перенести экраны в отдельный React/Vite клиент.
